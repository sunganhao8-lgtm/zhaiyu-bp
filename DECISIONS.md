## 🆕 2026-06-26 会议核心决议摘要

### DEC-012: 启动资金 12 万

**背景**：原 BP 写 22 万启动，会议逐项核算后发现无转让费、装修只需 1 万，实际启动远低于之前预估。

**决策**：
- 总启动资金 **12 万**（创始人A 5.1 万 / 创始人B 3.9 万 / 创始人C 3 万）
- 无转让费，仅付房租（年租 8 万 + 押金 8 千）
- 装修预算 1 万（粉刷 + 招牌）

**影响**：
- 每人出资约 3-4 万，门槛大幅降低
- 旧版所有"22 万/1.57 万/24-32 月"数字全部作废

### DEC-013: 员工无底薪

**背景**：前期现金流紧张，无法承担固定工资。

**决策**：
- 核心团队无底薪，盈利后按股分红
- 日常收入走"抽成（30%/50%）+ 分红"模式
- **老板无底薪**，同规则

### DEC-014: 股份结构

**决策**：
- 创始人A 42.5%（出资 5.1 万 + 赠送后调整）
- 创始人B 31%（出资 3.9 万 + 赠送后调整）
- 创始人C 24%（出资 3 万 + 赠送后调整）
- 店长D 5%（人力股，三位股东各让渡部分）

### DEC-015: 业务聚焦

**决策**：核心现金流只放宠物洗护、寄养、美容、宠物用品零售；上门服务作为增值业务验证；电竞方向由 DEC-024 单独定义；减脂餐、咖啡、AI 培训、活动策划等进入观察池，未验证前不计入主模型。

### DEC-016: 分润机制

**决策**："谁干谁拿钱"。普通业务抽成 30%，高耗时业务 50%。盈利后按股分红。股权永久绑定，后续交易需 >50% 决议通过。

### DEC-017: 营收目标

**决策**：
- 日经营贡献目标 **750 元**，日固定成本 **328 元**
- 当前预测日经营贡献 **699.5 元**（差 50.5 元/天）
- 目标 **12 个月** 回本

### DEC-024: 电竞房样板间定位

**背景**：补充确认门店装修时需要划出电竞房区域，既服务合伙人工作娱乐，也用于展示电竞房好物和上门搭建服务转化。

**决策**：
- 保留 5-8㎡ 干区作为**电竞房样板间**；
- 它不是独立收费电竞房，也不是游戏厅；
- 它承担合伙人工作娱乐、内容拍摄、桌搭好物展示、客户到店体验、上门电竞房搭建转化；
- 财务模型不按日散客流水计算，而按**曝光 → 咨询 → 到店看样板 → 上门方案成交 → 周边带货**漏斗计算。

### DEC-025: 业务分层与合规红线

**决策**：所有业务按五层管理：

| 层级 | 业务 |
|---|---|
| 首期必做 | 宠物洗护、美容、寄养、用品零售 |
| 样板展示 | 电竞房样板间、上门桌搭/电竞房搭建 |
| 验证后做 | 上门喂养、代理送医、宠物社群活动 |
| 观察池 | 减脂餐、咖啡/果茶、AI 培训、情侣/单身活动 |
| 暂不做或需资质 | 疫苗注射、宠物诊疗、酒馆、食品餐饮、学生自习收费、游戏厅式经营 |

### DEC-026: 抽成、岗位和分红分离

**决策**：
- “谁干谁拿钱”只适用于明确 owner 的具体服务项目；
- 股东分红、业务抽成、岗位补贴分开记账；
- 不建议日结分红，建议月度结账、季度分红；
- 分红前保留至少 3 个月安全资金。

### DEC-027: 客户发言档案作为核心增长引擎

**背景**：宅域不应只是“宠物 + 电竞 + 未来很多业务”的罗列。用户明确提出：每一位客户的发言都是洞悉需求的原材料。服务的作用是破开陌生客户对新品牌的防备，真正的转化来自持续倾听。

**决策**：
- 宅域统一主线升级为：**服务破冰 → 客户发言档案 → 需求方案化 → 产品化复购**；
- 宅咪和宅电只是首期两个高信任入口，不是项目边界；
- 每次服务、咨询、吐槽、犹豫、预算、空间照片和宠物情况都要进入客户发言档案；
- 每周复盘档案，形成高频需求 TOP10、内容选题、方案套餐、低库存商品和上门服务 SOP；
- 新业务必须从客户发言档案中反复出现的需求里长出来，而不是只靠合伙人脑暴。

**影响**：
- 首页、BP、合伙人沟通统一表达“客户发言是需求原材料”；
- 自媒体不只是宣传，而是需求测试工具；
- 门店不只是经营场地，而是信任发生器、样板间和需求采集点；
- 合伙人的价值不只是执行服务，还包括倾听、记录、复盘和产品化能力。

### DEC-028: 实体信任锚点与投资人异议话术

**背景**：一位看过计划书的外部反馈者连续追问“怎么落地，是实体店还是平台”“后期怎么运作”“视频不够好怎么办”“客户为什么敢让你上门”。这些问题代表投资人最容易卡住的核心疑虑：信任、获客、转化和平台化节奏。

**决策**：
- 对外统一回答：宅域不是先做空的平台或 App，而是先做实体信任锚点；
- 门店承担安全感、可追责、体验展示、内容拍摄和售后据点；
- 线上账号承担获客放大，但不把爆款视频作为唯一增长假设；
- 服务承担破冰和真实案例沉淀，长期利润来自商品选品、套餐方案、上门改造、复购和转介绍；
- 投资人沟通必须围绕 5 个可验证指标：有效咨询数、到店体验数、接送洗护成交数、商品附加率、复购/转介绍数。

**影响**：
- BP 首页新增“真实投资人对话提炼 · 6 个标准回答”；
- 新增 `docs/investor-objection-playbook.md` 作为投资人话术库；
- 融资表达从“我要做平台”改为“先验证实体信任锚点，再把档案、SOP、方案库和商品库平台化”。

## 工作流

1. **新决议** → 记录在会议笔记第 2 段 → 同步到 `data/decisions.yaml`
2. **冲突决议** → 强制在会议笔记第 3 段写冲突追踪
3. **修改 BP** → 先改 `data/facts.yaml` → 改 `bp.html` / `bps/store-front/bp.html/bp.md` → 跑 `tools/verify.py` 兜底
4. **新增业务** → 先查客户发言档案中是否出现重复需求 → 再做 MVP
5. **对外融资沟通** → 先引用 `docs/investor-objection-playbook.md`，确保实体店/平台/获客/信任/盈利回答一致
