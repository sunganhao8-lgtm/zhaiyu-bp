## 这是什么

宅域（zhaiyu）是一个位于**临港泥城**的社区门店创业项目。首期以**宠物洗护/美容/寄养**和**电竞房样板间**启动，但真正的主线不是“多开几个业务”，而是：

**用可信服务破冰，把每一位客户的发言、抱怨、偏好和预算沉淀成客户发言档案，再把高频家庭需求做成方案、产品和复购。**

宅咪解决宠物洗护与日常养护痛点，宅电表达理想居住环境。它们共同服务同一个目标：让宅域成为年轻人宅家生活需求的长期入口。

这个仓库是项目的**知识库**——包括商业计划书、会议记录、财务模型、合作着陆页、执行方案、以及保证文档一致性的工具。

## 核心增长模型

| 环节 | 作用 | 产出 |
|---|---|---|
| 服务破冰 | 用洗护、接送、桌搭诊断等低门槛服务破开客户对新品牌的防备 | 第一次信任、真实对话 |
| 发言入库 | 记录客户原话、生活场景、预算、犹豫点、宠物和居住信息 | 客户发言档案 |
| 需求归类 | 每周复盘高频抱怨和重复需求 | 需求 TOP10、内容选题 |
| 方案打样 | 用门店样板间、低库存 SKU、上门小单验证 | 可成交方案 |
| 产品化复购 | 把反复出现的需求做成套餐、好物清单、SOP、会员权益 | 毛利、复购、转介绍 |

## 为什么这样设计

这个项目的知识库记录了从立项到现在的所有**决策轨迹**，目的是解决两个常见问题：

1. **改文档漏改** — 修改 A 文件时 B 文件忘记同步。`data/facts.yaml` 是唯一事实源，`tools/verify.py` 可自动扫描不一致。
2. **新旧会议冲突** — 新会议的决定和之前定好的事项冲突。每篇会议笔记的「冲突追踪」段（第 3 段）**强制**记录与哪些历史决议冲突。
3. **业务发散** — 宠物、电竞、饮品、活动、AI 培训等想法必须分层管理：首期必做 / 样板展示 / 验证后做 / 暂不做或需资质。
4. **只凭灵感做业务** — 新业务不能只靠创始人脑暴，必须来自客户发言档案里的重复需求。
